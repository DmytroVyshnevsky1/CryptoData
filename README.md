
# Тестовое задание

Для получения данных я использовал **CoinAPI**.

## Описание задачи

В техническом задании было указано:

> "Приложение должно обладать функционалом периодического обновления курсов валют с реальной биржи."

Как такового смысла хранить данные о курсе нет, ведь они и так хранятся на серверах поставщика данных. Однако мы ограничены в количестве запросов, поэтому мне показалось, что лучше использовать кеширование запросов. Грамотная стратегия кеширования позволила бы значительно сократить количество запросов на серверы поставщика данных. Конечно, стратегия кеширования реализована очень примитивно в моем приложении, так как это тестовое задание :)

## Запуск приложения

Для запуска приложения используйте команду:

```sh
make up
```

## Тестовый клиент

Доступен по адресу: [http://localhost:3000/](http://localhost:3000/)

## Пример запроса на API

Данные можно получать с периодичностью в час и в 5 секунд. Пример запроса на API:

```http
http://127.0.0.1:8080/api/rates?base=BTC&quote=USD&period_id=1HRS&limit=1000
```
